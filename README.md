# REST API для управления клиентами (CRM)

Серверная часть реализована на Node.js (Express) и предоставляет REST API для работы с базой данных клиентов. Клиентская часть - статические файлы (HTML, CSS, JS), которые отдаются сервером и обеспечивают пользовательский интерфейс.

## Требования

- Node.js версии 12 или выше
- npm (устанавливается вместе с Node.js)

## Установка и запуск

1. Склонируйте репозиторий и перейдите в папку проекта.
2. Установите зависимости:
   ```bash
   npm install
   ```
3. Запустите сервер в production-режиме:
   ```bash
   npm start
   ```
   или в режиме разработки с автоматическим перезапуском:
   ```bash
   npm run dev
   ```
4. Сервер будет доступен по адресу `http://localhost:3000` (порт может быть изменён через переменную окружения `PORT`).
5. Файл базы данных `db.json` создаётся автоматически в корне проекта (путь можно изменить через `DB_FILE`).

## Тестирование

Для запуска тестов (Jest + Supertest):
```bash
npm test
```

## API

Базовый префикс всех методов: `/api/clients`

Все запросы, ожидающие тело, должны передавать данные в формате JSON. Ответы сервера также возвращаются в виде JSON.

### Методы

#### Получить список клиентов
```
GET /api/clients
```
**Параметры строки запроса:**
- `search` (необязательный) – поисковый запрос.

**Поведение поиска:**
- Поисковый запрос разбивается на отдельные слова (по пробельным символам).
- Каждое слово должно присутствовать либо в полном имени клиента (фамилия + имя + отчество), либо в значении любого из его контактов.
- **Дополнительная фильтрация по алфавиту:**
  - Если поисковый запрос содержит хотя бы одну кириллическую букву, в результатах остаются только клиенты, в полном имени которых есть кириллица.
  - Если поисковый запрос **не содержит** кириллицы (только латиница, цифры, символы), в результатах остаются только клиенты, в полном имени которых **нет** кириллицы.
  - Это позволяет различать поиск по русскоязычным и англоязычным именам.

**Пример:**
```
GET /api/clients?search=Иван
GET /api/clients?search=john
```

**Ответ:** массив объектов клиентов (статус 200).

---

#### Создать нового клиента
```
POST /api/clients
```
Тело запроса должно содержать объект клиента (обязательные поля: `name`, `surname`).

**Заголовки ответа:**
- `Location` – URL созданного ресурса (например, `/api/clients/123`).
- `Access-Control-Expose-Headers: Location`

**Статус ответа:** 201 Created.

**Тело ответа:** объект созданного клиента (с автоматически сгенерированными `id`, `createdAt`, `updatedAt`).

---

#### Получить клиента по ID
```
GET /api/clients/:id
```
**Параметры пути:** `id` – идентификатор клиента.

**Ответ:** объект клиента (статус 200).

**Ошибка:** 404, если клиент не найден.

---

#### Обновить данные клиента
```
PATCH /api/clients/:id
```
Тело запроса может содержать любые поля клиента (кроме `id`, `createdAt`, `updatedAt`). Непереданные поля остаются неизменными.

**Ответ:** объект обновлённого клиента (статус 200).

**Ошибки:** 404 (клиент не найден), 422 (некорректные данные, например пустое имя).

---

#### Удалить клиента
```
DELETE /api/clients/:id
```
**Ответ:** пустой объект `{}` (статус 200).

**Ошибка:** 404, если клиент не найден.

---

## Структура объекта клиента

```javascript
{
  // Уникальный идентификатор (строка). Генерируется автоматически, не изменяется.
  id: "123",

  // Дата и время создания (ISO 8601). Автоматически, не изменяется.
  createdAt: "2021-02-03T13:07:29.554Z",

  // Дата и время последнего изменения (ISO 8601). Обновляется автоматически.
  updatedAt: "2021-02-03T13:07:29.554Z",

  // Обязательное поле. Имя клиента.
  name: "Василий",

  // Обязательное поле. Фамилия клиента.
  surname: "Пупкин",

  // Необязательное поле. Отчество клиента.
  lastName: "Васильевич",

  // Необязательное поле. Массив контактов.
  contacts: [
    {
      type: "Телефон",   // строка, обязательное поле для контакта
      value: "+71234567890"  // строка, обязательное поле для контакта
    },
    {
      type: "Email",
      value: "abc@xyz.com"
    }
  ]
}
```

**Примечания:**
- Поля `name` и `surname` не могут быть пустыми строками.
- Если в теле запроса присутствуют контакты, то каждый объект в массиве `contacts` должен иметь непустые `type` и `value`. В противном случае будет ошибка валидации.
- Сервер сам заполняет `id`, `createdAt`, `updatedAt`. Эти поля игнорируются при создании и обновлении.

## Статусы ответов и ошибки

| Код | Описание |
|-----|-----------|
| 200 | Запрос успешно обработан (GET, PATCH, DELETE). |
| 201 | Клиент успешно создан (POST). |
| 404 | Ресурс не найден (неверный ID или метод). |
| 422 | Ошибка валидации данных в теле запроса. |
| 500 | Внутренняя ошибка сервера. |

**Формат ошибки 422:**
```javascript
{
  "errors": [
    {
      "field": "name",
      "message": "Не указано имя"
    },
    {
      "field": "contacts",
      "message": "Не все добавленные контакты полностью заполнены"
    }
  ]
}
```

## Клиентская часть

Сервер также отдаёт статические файлы из папки `public`. Откройте в браузере `http://localhost:3000`, чтобы увидеть веб-интерфейс для работы с клиентами. Он предоставляет:
- Таблицу со списком клиентов.
- Поиск с автодополнением (учитывает поведение серверного поиска).
- Кнопку добавления нового клиента (модальное окно).
- Редактирование и удаление клиентов.

## Файл базы данных

По умолчанию данные хранятся в файле `db.json` в корне проекта. Вы можете изменить путь через переменную окружения `DB_FILE`. При первом запуске файл создаётся автоматически с содержимым `[]`.

## Переменные окружения

- `PORT` – порт, на котором запускается сервер (по умолчанию `3000`).
- `DB_FILE` – путь к файлу базы данных (по умолчанию `./db.json`).

## Клиентский интерфейс (подробно)

Веб-интерфейс представляет собой одностраничное приложение (SPA) на чистом JavaScript, HTML и CSS. Все данные загружаются и отправляются через REST API, описанное выше.

### Основные возможности

- **Таблица клиентов**  
  Отображает список клиентов с колонками: ID, ФИО, дата/время создания, дата/время последнего изменения, контакты, действия.

- **Сортировка**  
  Клик по заголовкам колонок `ID`, `Фамилия, имя, отчество`, `Дата и время создания`, `Последнее изменение` сортирует таблицу. Повторный клик меняет направление (по возрастанию/убыванию). Активная сортировка визуально обозначена стрелкой в заголовке.

- **Поиск и автодополнение**  
  Поле поиска в шапке отправляет запросы на сервер с параметром `search`. При вводе текста через 300 мс выполняется поиск, результаты отображаются в таблице.  
  **Автодополнение:** под полем ввода появляется выпадающий список с уникальными полными именами клиентов (до 10 вариантов). Выбор подсказки заполняет поле поиска и сразу выполняет фильтрацию.  
  *Техническая деталь:* при быстром вводе предыдущие запросы отменяются через `AbortController`, чтобы избежать гонки ответов.

- **Добавление клиента**  
  Кнопка *«Добавить клиента»* открывает модальное окно с формой.  
  - Поля *Фамилия* и *Имя* обязательны (отмечены звёздочкой).  
  - Можно добавить до 10 контактов (тип: Телефон, Email, VK, Facebook, Другое).  
  - Для каждого контакта нужно заполнить оба поля, иначе форма не отправится.  
  - Кнопка *«Сохранить»* отправляет `POST /api/clients`.  
  - После успешного создания таблица обновляется, модальное окно закрывается.

- **Редактирование клиента**  
  Нажать *«Изменить»* в строке клиента или перейти по прямой ссылке вида `#client-<id>` (например, `http://localhost:3000/#client-5`).  
  Открывается модальное окно с предзаполненными данными.  
  - Можно изменить любые поля, включая контакты.  
  - Если изменения не были внесены, кнопка *«Сохранить»* покажет ошибку.  
  - Кнопка *«Удалить клиента»* (в модальном окне режима редактирования) открывает диалог подтверждения.

- **Удаление клиента**  
  Через кнопку *«Удалить»* в строке таблицы или из модального окна редактирования.  
  Появляется модальное окно с вопросом подтверждения. После подтверждения клиент удаляется, таблица обновляется.

- **Просмотр контактов**  
  В колонке «Контакты» выводятся иконки (телефон, email, VK, Facebook, другое).  
  - При наведении на иконку появляется тултип с типом и значением контакта.  
  - Клик по иконке открывает соответствующее действие:  
    - телефон → `tel:` ссылка (звонок),  
    - email → `mailto:` ссылка,  
    - остальные (VK, Facebook, другое) → ссылка на указанный адрес (открывается в новой вкладке).

- **Копирование ссылки на клиента**  
  Двойной клик по любой строке таблицы копирует в буфер обмена прямую ссылку на редактирование этого клиента (формат: `http://.../#client-<id>`).  
  Появляется временное всплывающее уведомление *«Ссылка скопирована»*.

- **Прямые ссылки (глубокие ссылки)**  
  Если открыть URL с хешем `#client-<id>`, интерфейс автоматически откроет модальное окно редактирования соответствующего клиента. Это позволяет делиться ссылками на конкретную карточку.

- **Индикация загрузки**  
  Во время загрузки списка клиентов в области таблицы отображается анимированный спиннер. При отправке форм в модальном окне также показывается локальный прелоадер, блокирующий повторные действия.

- **Валидация форм**  
  - На стороне клиента проверяется заполнение имени, фамилии, корректность контактов.  
  - При ошибках поля подсвечиваются красным, а над кнопками выводится список ошибок.  
  - Ошибки с сервера (статус 422) также отображаются пользователю в том же блоке.

- **Обработка ошибок сети и сервера**  
  При проблемах с загрузкой данных или отправке формы пользователь видит понятное сообщение об ошибке в интерфейсе.

### Структура клиентских файлов

- `public/index.html` – основная страница, подключает CSS и JS.  
- `public/index.js` – вся логика приложения: работа с API, управление DOM, модальные окна, сортировка, поиск, автодополнение, хеш-навигация.  
- `public/css/` – стили, разделённые по компонентам:
  - `normolize.css` – сброс стилей.
  - `index.css` – импорты всех CSS-модулей.
  - `blocks/header/`, `blocks/main/`, `blocks/modal/` – стили для соответствующих частей.
  - CSS-переменные, анимации (`keyframes/__animations.css`), иконки контактов (как фоновые изображения SVG).

### Требования к окружению для работы клиентской части

- Современный браузер с поддержкой ES6 (Fetch API, AbortController, CSS Grid/Flex).  
- Сервер должен быть запущен и доступен по адресу, указанному в `index.js` (по умолчанию `http://localhost:3000`).  
- Для корректной работы автодополнения и поиска требуется поддержка `AbortController`.

## Особенности реализации (дополнительно)

- **Сортировка** выполняется на клиенте после получения данных, чтобы снизить нагрузку на сервер.  
- **Автодополнение** использует тот же endpoint `/api/clients?search=...`, но отображает только уникальные полные имена.  
- **Хеш-навигация** реализована через `window.location.hash` и событие `hashchange`. При загрузке страницы с хешем модальное окно открывается автоматически.  
- **Копирование ссылки** работает через Clipboard API с fallback на `alert`.  
- **Модальные окна** реализованы через динамически создаваемые `div` с классами `.modal` и анимацией появления. Закрытие по клику на фон, кнопку «крестик» или клавишу ESC (не показано в коде, но можно добавить).  
- **Плейсхолдеры в форме** реализованы с помощью `fake-placeholder` и JavaScript-класса `.form-group--filled` для анимированного подъёма подписи над полем ввода.  
- **Контакты** динамически добавляются/удаляются с помощью кнопок «Добавить контакт» и крестика в каждой строке.
